# insight-frameworks.md — 분석력·인사이트 프레임워크 v0.9

> 목적: 챕터가 '소스 요약'에 머물지 않고 **분석과 통찰**이 되게 한다.
> Phase 2(구조화)와 챕터 초안 작성 시 아래 절차를 적용하고,
> quality_rubric.md 의 #8(인사이트 깊이) 채점 기준이 이를 검사한다.

---

## 1. So-What 사다리 (모든 KEY MESSAGE의 생성 절차)

| 단계 | 질문 | 예시 |
|---|---|---|
| L1 사실 | 무엇이 일어났나 | "A사가 B 모델을 공개" |
| L2 의미 | 그래서 무엇이 달라지나 | "추론 비용이 1/10로 하락" |
| L3 함의 | 누가 이기고 누가 지나 (2차 효과) | "API 재판매 사업모델 붕괴, 온디바이스 진영 수혜" |
| L4 행동 | 독자는 무엇을 해야 하나 | "고객사는 멀티벤더 계약으로 전환 검토" |

- KEY MESSAGE(Zone2, H2 헤드라인)는 **L2 이상**에서 뽑는다. L1 사실 나열형 금지.
- 챕터당 최소 1개는 L3(2차 효과)까지 도달.
- Executive Summary의 Key Message는 보고서 전체를 관통하는 L3~L4 명제 1~2문장.

## 2. 수치 다루기 (분석의 신뢰 기반)

- **비교 없는 수치 금지**: 절대값만 쓰지 말고 전년 대비·경쟁사 대비·시장 평균 대비 중
  하나의 비교축을 붙인다. ("매출 3.2조 원" → "매출 3.2조 원, YoY +18%로 업계 평균 2배")
- **교차검증**: 핵심 수치(시장 규모·점유율)는 가능하면 2개 소스 대조. 충돌 시 병기.
- **단위 정규화**: 환율·회계연도·억/兆 단위를 본문 내 일관 유지 (claims.json note에 기준 기록).
- 모든 수치는 factcheck-pipeline.md 의 Claims Ledger에 등록.

## 3. 반론 내성 (Devil's Advocate를 챕터 안으로)

- 챕터의 핵심 주장마다 **가장 강한 반론 1개**를 스스로 생성해 본문에서 다룬다
  (반박하거나, 조건부로 인정). 반론 없이 일방 서술된 챕터는 rubric #8에서 감점.
- 마지막 Devil's Advocate 챕터는 이를 종합해 '이 전망이 틀리는 3가지 경로'를 다룬다.

## 4. 이해관계자 렌즈

핵심 사건마다 최소 3개 관점에서 함의를 검토: ①당사자(기업), ②경쟁자/대체재,
③수요자(고객·투자자·규제당국). 한 관점만 서술된 분석은 표면적이다.

## 5. 전망 작성 규칙

- 단기(1년 내)/중기(2~3년)/장기(5년+)를 구분하고 각각 **조건문**으로 쓴다:
  "X가 유지된다면 ~, 반면 Y가 일어나면 ~".
- 전망 수치는 전부 estimate 처리(추정 표기). 근거 외삽 방식을 1문장으로 명시.

## 6. 비유·예시 (쉬운 전달 — rubric #4)

- 챕터당 1개 이상, 일상 사물 비유 또는 역사적 유사 사례.
- 비유는 구조를 설명해야 한다 — 장식용 수사는 금지 형용사 정책과 동일하게 배제.
