# quality_rubric.md — LLM-judge 주관 품질 루브릭 (v0.9)

`verify_content.py` 가 **결정론적으로 잡히는 위반**(과장 형용사·bullet·한자 혼입 등)을
보고 `verify_claims.py` 가 **수치-출처 정합**을 본다면, 이 루브릭은 코드로 잡기 어려운
**주관적 품질**을 모델(=judge)이 채점하는 기준이다. judge 는 가능하면 작성자와 **다른
컨텍스트의 critic 에이전트**가 맡는다(동적 워크플로 Phase D — self-preferential bias 차단).
단일 컨텍스트 모드에서는 빌드 후 "비평가 모자"를 쓰고 채점한다.
산출은 `judge_verdict_{lang}.json`.

## 입력
`verify_content.py --judge-extract judge_in_{lang}.json` 로 덤프된 슬라이드/문서별 텍스트.

## 채점 기준 (각 0·1·2점, 합 16점 만점)

| # | 기준 | 0점 | 1점 | 2점 |
|---|------|-----|-----|-----|
| 1 | **KEY MESSAGE 완결성** | 명제 없음/단어 나열 | 있으나 모호 | 데이터를 관통하는 한 줄 통찰(40~100자) |
| 2 | **근거·로직** | 주장만 | 근거 일부 | 모든 주장에 수치·사례·출처 연결 |
| 3 | **산문체(입니다/합니다)** | 개조식 혼재 | 일부 이탈 | 3~4 단락 완전한 격식 산문 |
| 4 | **쉬운 전달(비유·예시)** | 전문용어 나열 | 예시 빈약 | 초등학생도 이해할 anecdote+전문성 양립 |
| 5 | **객관성(과장 배제)** | 화려한 형용사 다수 | 일부 | 객관·중립 서술 |
| 6 | **유기적 연결** | 단락 단절 | 느슨 | 하나의 완결된 이야기 흐름 |
| 7 | **TOC↔본문 내러티브 정합** | 제목과 내용 불일치 | 부분 | 챕터 약속과 내용 정확히 일치 |
| 8 | **인사이트 깊이 (So-What)** | 소스 요약·사실 나열(L1) | 의미 해석까지(L2) | 2차 효과·행동 함의(L3~L4) + 반론 1개 이상 처리 |

> #8 채점 근거는 `insight-frameworks.md` 의 So-What 사다리. KEY MESSAGE 가 전부
> "무엇이 있었다" 형이면 0점, "그래서 무엇이 달라진다"가 있으면 1점,
> "누가 이기고 지는가/무엇을 해야 하는가"와 반론 처리가 보이면 2점.

**시각화 메모(비차단)**: `visual-catalog.md` 매핑 대비 시각요소(표·플로우차트·차트)
활용을 summary 에 코멘트한다. 프로세스 설명이 줄글로만 길게 이어진 챕터에는 권장 fix 를
남기되, 시각화는 합격/불합격 판정에 **포함하지 않는다**(권장 가이드).

## 합격선
- **총점 ≥ 12 그리고 어떤 기준도 0점 없음** → `pass: true`
- 그 외 → `pass: false`, 0~1점 기준마다 fix 항목 생성

## judge 출력 형식 (gate_report.py 가 그대로 소비)
```json
{
  "pass": false,
  "score": 11,
  "summary": "KEY MESSAGE 약함, slide5 근거 부족, ch3 인사이트 L1 수준",
  "fixes": [
    {"type": "JUDGE_KEYMSG", "loc": "slide5",
     "detail": "KEY MESSAGE가 사실 나열에 그침",
     "fix": "Zone2를 '~로 시장이 재편되고 있음.' 형태의 통찰문으로 교체"},
    {"type": "JUDGE_INSIGHT", "loc": "ch3",
     "detail": "사실 요약만 있고 2차 효과 분석 없음",
     "fix": "So-What 사다리 L3 — 수혜/피해 주체와 행동 함의 단락 추가"}
  ]
}
```

## 루프 안에서의 위치
build → `verify_toc`(구조) → `verify_content`(규칙) → `verify_claims`(수치-출처)
→ **judge(이 루브릭)** → `gate_report`(합산).
gate_report 가 넷을 합쳐 `pass`/`fixes[]` 를 내면, 재시도 루프가 fixes 를 적용한다.
